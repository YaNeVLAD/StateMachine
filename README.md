# FSM & Grammar C++ Library

Современная C++20 header-only библиотека для работы с конечными автоматами, регулярными и
контекстно-свободными грамматиками, а также для построения лексических и синтаксических LL(1)-анализаторов.

---

## Модуль 1: Система Трейтов и Концептов (Ядро библиотеки)

Архитектура библиотеки использует трейты для абстрагирования алгоритмов от конкретных реализаций автоматов.
Чтобы алгоритмы библиотеки работали с вашим автоматом, необходимо специализировать следующие структуры:

### `fsm::state_machine_traits<T_StateMachine>`

*(Содержит только псевдонимы типов, методов нет)*

* `state_type` — тип, хранящий состояние автомата.
* `input_type` — тип входного символа.
* `output_type` — тип возвращаемого результата после перехода.

### `fsm::translation_traits<T_StateMachine>`

Описывает логику поиска переходов.

* **Шаблонные параметры**: `T_StateMachine` — тип вашего автомата.

| Метод               | Аргументы                                                | Возвращаемый тип   | Описание                                                                                       |
|:--------------------|:---------------------------------------------------------|:-------------------|:-----------------------------------------------------------------------------------------------|
| `find` (static)     | `const state_type& state`, `const input_type& input`     | `find_result_type` | Ищет переход для текущего состояния по входному символу. Возвращает итератор или спец. объект. |
| `is_valid` (static) | `const find_result_type& res`, `const state_type& state` | `bool`             | Проверяет, валиден ли найденный переход (например, `res != end()`).                            |
| `result` (static)   | `const find_result_type& res`                            | `result_type`      | Извлекает результат из найденного перехода (обычно ID следующего состояния).                   |

### `fsm::minimization_traits<T_StateMachine>`

Описывает интерфейс для алгоритма минимизации на основе классов эквивалентности.

* **Шаблонные параметры**: `T_StateMachine` — тип минимизируемого автомата.

| Метод                                 | Аргументы                                                                           | Возвращаемый тип          | Описание                                                                                       |
|:--------------------------------------|:------------------------------------------------------------------------------------|:--------------------------|:-----------------------------------------------------------------------------------------------|
| `get_all_state_ids` (static)          | `const state_type& state`                                                           | `std::vector<id_type>`    | Возвращает список всех ID состояний автомата.                                                  |
| `get_all_inputs` (static)             | `const state_type& state`                                                           | `std::vector<input_type>` | Возвращает алфавит (все возможные входы).                                                      |
| `get_next_state_id` (static)          | `const state_type& state`, `const id_type& current`, `const input_type& input`      | `id_type`                 | Возвращает ID состояния, в которое ведет переход.                                              |
| `are_0_equivalent` (static)           | `const state_type& state`, `const id_type& s1`, `const id_type& s2`                 | `bool`                    | Проверяет, эквивалентны ли состояния `s1` и `s2` на 0-м шаге (оба финальные/одинаковый выход). |
| `reconstruct_from_partition` (static) | `const T_StateMachine& original`, `const std::vector<std::set<id_type>>& partition` | `T_StateMachine`          | Собирает новый минимизированный автомат на основе итоговых классов эквивалентности.            |

## Модуль 2: Конечные Автоматы (FSM)

### `fsm::base_state_machine<T_Derived, T_Traits>`

Базовый класс (CRTP), реализующий цикл обработки входа.

* **Шаблонные параметры**: `T_Derived` — ваш класс автомата, `T_Traits` — специализация `state_machine_traits`.

| Метод / Конструктор | Аргументы                         | Возвращаемый тип    | Описание                                                            |
|:--------------------|:----------------------------------|:--------------------|:--------------------------------------------------------------------|
| **Конструктор**     | `const state_type& initial_state` | -                   | Инициализирует автомат начальным состоянием (копирование).          |
| **Конструктор**     | `state_type&& initial_state`      | -                   | Инициализирует автомат начальным состоянием (перемещение).          |
| `handle_input`      | `const input_type& input`         | `output_type`       | Выполняет переход по символу и возвращает результат перехода.       |
| `state`             | -                                 | `const state_type&` | Возвращает константную ссылку на текущее полное состояние автомата. |

### `fsm::base_recognizer<T_State>` (Псевдоним: `fsm::recognizer`)

Распознаватель, имеющий возможность анализировать входную цепочку и возвращать `bool` (достигнуто ли финальное
состояние).

* **Шаблонные параметры**: `T_State` — структура состояния (по умолчанию `recognizer_state`).

| Метод / Конструктор | Аргументы                                           | Возвращаемый тип  | Описание                                                                               |
|:--------------------|:----------------------------------------------------|:------------------|:---------------------------------------------------------------------------------------|
| **Конструктор**     | `const T_State& / T_State&& initial_state`          | -                 | Создает распознаватель из стейт-объекта.                                               |
| **Конструктор**     | `const mealy_machine&`, `const std::set<state_id>&` | -                 | Конвертирует автомат Мили в распознаватель.                                            |
| `handle_input`      | `Container&& inputs` (шаблон)                       | `bool`            | Обрабатывает коллекцию символов. Возвращает `true`, если итог — в финальном состоянии. |
| `is_deterministic`  | -                                                   | `bool`            | Возвращает флаг детерминированности автомата.                                          |
| `from_dot` (static) | `const std::string& filename`                       | `base_recognizer` | Загружает автомат из файла формата Graphviz DOT.                                       |

### Свободные функции (Алгоритмы)

| Функция            | Аргументы                                      | Возвращаемый тип | Описание                                                                   |
|:-------------------|:-----------------------------------------------|:-----------------|:---------------------------------------------------------------------------|
| `fsm::determinize` | `const recognizer& rec`                        | `recognizer`     | Преобразует НКА (NFA) в ДКА (DFA).                                         |
| `fsm::minimize`    | `const T_StateMachine& machine`                | `T_StateMachine` | Минимизирует количество состояний детерминированного автомата.             |
| `fsm::recognize`   | `T_Recognizer& rec`, `Args&&... inputs`        | `output_type`    | Прогоняет цепочку входов через автомат без изменения его исходного стейта. |
| `fsm::dot`         | `std::ostream& os`, `const T_Machine& machine` | `void`           | Экспортирует граф автомата в поток в формате DOT.                          |

<details>
<summary><b>Пример использования (Нажмите, чтобы развернуть)</b></summary>

```cpp
#include "fsm/fsm.hpp"
#include <iostream>

int main() {
    fsm::recognizer_state nfa_state;
    nfa_state.state_ids = {"q0", "q1"};
    nfa_state.initial_state_id = "q0";
    nfa_state.final_state_ids = {"q1"};
    nfa_state.transitions.insert({{"q0", "a"}, "q0"});
    nfa_state.transitions.insert({{"q0", "a"}, "q1"}); // Недетерминизм
    nfa_state.is_deterministic = false;

    fsm::recognizer nfa(std::move(nfa_state));
    fsm::recognizer min_dfa = fsm::minimize(fsm::determinize(nfa));

    std::cout << fsm::recognize(min_dfa, "aaaa") << "\n"; // 1 (true)
    return 0;
}
```

</details>

## Модуль 3: Регулярные Выражения и Лексер

### `fsm::base_regex<T_Parser, T_Builder>` (Псевдоним: `fsm::regex`)

Компилятор регулярных выражений в конечный автомат.

* **Шаблонные параметры**: Внутренние парсеры и билдеры AST.

| Метод / Конструктор | Аргументы                                                    | Возвращаемый тип | Описание                                                                  |
|:--------------------|:-------------------------------------------------------------|:-----------------|:--------------------------------------------------------------------------|
| **Конструктор**     | `const std::string& expr`, `bool compile_immediately = true` | -                | Принимает строку regex и парсит её в AST (по умолчанию сразу строит NFA). |
| `compile`           | -                                                            | `recognizer`     | Возвращает готовый распознаватель для заданного выражения.                |

### `fsm::lexer<T_TokenType, T_Matcher>`

Универсальный лексический анализатор.

* **Шаблонные параметры**: `T_TokenType` — перечисление или тип токена (enum), `T_Matcher` — движок (по умолчанию
  `fsm_regex_matcher`).

| Метод / Конструктор | Аргументы                                                          | Возвращаемый тип       | Описание                                                                             |
|:--------------------|:-------------------------------------------------------------------|:-----------------------|:-------------------------------------------------------------------------------------|
| **Конструктор**     | `std::string_view source`                                          | -                      | Инициализирует лексер исходным кодом.                                                |
| `add_rule`          | `const std::string& expr`, `T_TokenType type`, `bool skip = false` | `lexer&`               | Добавляет правило. Если `skip=true`, токен будет проигнорирован (например, пробелы). |
| `change_source`     | `std::string_view source`, `bool with_clear = true`                | `lexer&`               | Меняет исходный текст. По умолчанию удаляет старые правила (`with_clear`).           |
| `peek`              | -                                                                  | `std::optional<token>` | Возвращает следующий токен без смещения курсора чтения.                              |
| `next`              | -                                                                  | `std::optional<token>` | Читает и возвращает следующий токен. Смещает курсор.                                 |
| `tokenize`          | -                                                                  | `std::vector<token>`   | Обрабатывает весь текст и возвращает список всех токенов.                            |

<details>
<summary><b>Пример использования (Нажмите, чтобы развернуть)</b></summary>

```cpp
#include "fsm/lexer.hpp"
#include <iostream>

enum class Token { Ident, Number, Space };

int main() {
    fsm::lexer<Token> lexer("var 42");
    lexer.add_rule("[a-z]+", Token::Ident)
         .add_rule("[0-9]+", Token::Number)
         .add_rule("[ \t]+", Token::Space, true); // Пропуск

    for (const auto& tok : lexer.tokenize()) {
        std::cout << "Lexeme: " << tok.lexeme << " at col " << tok.column << "\n";
    }
    return 0;
}
```

</details>

## Модуль 4: Контекстно-свободные Грамматики (CFG)

### `fsm::basic_cfg<T_Symbol, T_Comparator>` (Псевдоним: `fsm::cfg`)

Хранилище контекстно-свободной грамматики.

* **Шаблонные параметры**: `T_Symbol` — тип символа (по умолчанию `std::string`), `T_Comparator` — функтор сравнения.

| Метод / Конструктор | Аргументы                                                  | Возвращаемый тип  | Описание                                               |
|:--------------------|:-----------------------------------------------------------|:------------------|:-------------------------------------------------------|
| **Конструктор**     | Пустой или (non_terminals, terminals, rules, start_symbol) | -                 | Инициализирует пустую грамматику или из набора данных. |
| `add_rule`          | `const cfg_rule<T_Symbol>& new_rule`                       | `void`            | Добавляет правило вида `A -> B C`.                     |
| `set_start_symbol`  | `T_Symbol new_start`                                       | `void`            | Устанавливает стартовый нетерминал.                    |
| `non_terminals`     | -                                                          | `const std::set&` | Возвращает набор нетерминалов.                         |
| `terminals`         | -                                                          | `const std::set&` | Возвращает набор терминалов.                           |
| `rules`             | -                                                          | `const std::set&` | Возвращает набор правил грамматики.                    |
| `print`             | `std::ostream& os = std::cout`                             | `void`            | Печатает грамматику в удобном читаемом формате.        |

### Свободные функции и Трансформации

Трансформации применяются с помощью оператора `|` (конвейер), аналогично `std::views`.

| Функция / Трансформация  | Аргументы          | Возвращаемый тип | Описание                                                   |
|:-------------------------|:-------------------|:-----------------|:-----------------------------------------------------------|
| `fsm::cfg_load`          | `std::istream& in` | `basic_cfg`      | Парсит текст вида `A -> B c \| ε` и возвращает объект CFG. |
| `remove_epsilon_rules`   | -                  | Трансформация    | Удаляет ε-правила (с сохранением языка).                   |
| `remove_unit_rules`      | -                  | Трансформация    | Удаляет цепные правила вида `A -> B`.                      |
| `remove_useless_symbols` | -                  | Трансформация    | Удаляет недостижимые и непроизводящие символы.             |
| `to_chomsky_normal_form` | -                  | Трансформация    | Приводит грамматику к НФ Хомского.                         |
| `remove_left_recursion`  | -                  | Трансформация    | Устраняет прямую и косвенную левую рекурсию.               |
| `left_factor`            | -                  | Трансформация    | Выполняет левую факторизацию (вынос общего префикса).      |

<details>
<summary><b>Пример использования (Нажмите, чтобы развернуть)</b></summary>

```cpp
#include "fsm/cfg.hpp"
#include <sstream>
#include <iostream>

int main() {
    std::istringstream input("S -> A B | ε \nA -> a A | a \nB -> b \n");
    auto cfg = fsm::cfg_load(input);

    using namespace fsm::transforms;
    auto optimized_cfg = cfg | remove_epsilon_rules | remove_unit_rules | to_chomsky_normal_form;

    optimized_cfg.print(std::cout);
    return 0;
}
```

</details>

## Модуль 5: LL(1) Синтаксический Анализ

Модуль предоставляет инструменты для автоматического вычисления множеств `FIRST` и `FOLLOW`, построения таблицы
предсказывающего анализатора и проверки принадлежности цепочек токенов заданному языку.

### `fsm::ll1::table_builder<T_Symbol, T_Compare>`

Вычисляет множества FIRST/FOLLOW и строит управляющую таблицу LL(1)-анализатора из контекстно-свободной грамматики.

* **Шаблонные параметры**: `T_Symbol` — тип символа грамматики (по умолчанию работает с тем же типом, что и
  `basic_cfg`), `T_Compare` — компаратор.

| Метод / Конструктор       | Аргументы                                                                | Возвращаемый тип  | Описание                                                                                                                      |
|:--------------------------|:-------------------------------------------------------------------------|:------------------|:------------------------------------------------------------------------------------------------------------------------------|
| **Конструктор**           | `const basic_cfg& grammar`, `const T_Symbol& eps`, `const T_Symbol& eof` | -                 | Привязывает грамматику, задает символ эпсилон (`ε`) и маркер конца цепочки (например, `$`).                                   |
| `with_epsilon`            | `const T_Symbol& epsilon`                                                | `table_builder&`  | Задает символ, обозначающий пустое правило (ε-переход).                                                                       |
| `with_end_marker`         | `const T_Symbol& end_marker`                                             | `table_builder&`  | Задает маркер конца входной цепочки (EOF).                                                                                    |
| `with_collision_strategy` | `collision_strategy strategy`                                            | `table_builder&`  | Стратегия разрешения конфликтов при неоднозначностях грамматики: `throw_exception` (по умолчанию), `keep_first`, `keep_last`. |
| `on_warning`              | `std::function<void(const std::string&)> cb`                             | `table_builder&`  | Коллбэк, вызываемый для вывода подробного анализа конфликта (FIRST/FOLLOW), если стратегия не `throw_exception`.              |
| `build`                   | `const T_Formatter& formatter = {}`                                      | `table<T_Symbol>` | Вычисляет FIRST/FOLLOW и возвращает готовую LL(1)-таблицу.                                                                    |

### `fsm::ll1::table<T_Symbol, T_Compare>`

Контейнер, хранящий построенную матрицу (таблицу предсказаний).

* **Шаблонные параметры**: `T_Symbol` — тип символа, `T_Compare` — компаратор (по умолчанию `std::less<T_Symbol>`).

| Метод / Конструктор | Аргументы                                                                  | Возвращаемый тип                                         | Описание                                                                                                     |
|:--------------------|:---------------------------------------------------------------------------|:---------------------------------------------------------|:-------------------------------------------------------------------------------------------------------------|
| **Конструктор**     | `const basic_cfg&`, `const T_Symbol& eps`, `const T_Symbol& eof`           | -                                                        | Инициализирует пустую таблицу, запоминая спецсимволы.                                                        |
| `has_rule`          | `const T_Symbol& lhs`, `const T_Symbol& terminal`                          | `bool`                                                   | Проверяет, есть ли правило для пары `[Нетерминал, Терминал]`.                                                |
| `at` / `operator[]` | `const T_Symbol& lhs`, `const T_Symbol& terminal`                          | `const rule_type&`                                       | Возвращает правило перехода по нетерминалу и входному токену. Бросает `std::out_of_range`, если правила нет. |
| `find`              | `const T_Symbol& lhs`, `const T_Symbol& terminal`                          | `std::optional<std::reference_wrapper<const rule_type>>` | Безопасный поиск правила. Возвращает `std::nullopt`, если ячейка пуста.                                      |
| `add_entry`         | `const T_Symbol& lhs`, `const T_Symbol& terminal`, `const rule_type& rule` | `table&`                                                 | Добавляет правило. Бросает исключение при коллизии, если добавляется отличное от существующего правило.      |
| `set_entry`         | `const T_Symbol& lhs`, `const T_Symbol& terminal`, `const rule_type& rule` | `table&`                                                 | Принудительно устанавливает или перезаписывает правило в ячейке.                                             |
| `entries`           | -                                                                          | `const storage_t&`                                       | Возвращает внутреннюю структуру таблицы `map<T, map<T, rule>>`.                                              |
| `empty`             | -                                                                          | `bool`                                                   | Проверяет, пуста ли таблица.                                                                                 |

### Свободные функции LL(1) Анализатора

| Функция                 | Аргументы                                                                                        | Возвращаемый тип | Описание                                                                                                                                                                                                              |
|:------------------------|:-------------------------------------------------------------------------------------------------|:-----------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `fsm::ll1::accepts`     | `table`, `start_symbol`, `epsilon`, `eof`, `input_sequence`, `err_out`, `trace_out`, `formatter` | `bool`           | Выполняет табличный LL(1)-парсинг (эмуляция автомата с магазинной памятью). Возвращает `true`, если входная цепочка принадлежит языку. Может выводить подробную трассировку стека в `trace_out` и ошибки в `err_out`. |
| `fsm::ll1::print_table` | `table`, `std::ostream&`, `settings`, `formatter`                                                | `void`           | Красиво форматирует и выводит LL(1)-таблицу. Поддерживает режимы: `rules_list`, `compiled_table`, `detailed_table`.                                                                                                   |

### Пример использования: От грамматики к парсеру

```cpp
#include "fsm/fsm.hpp"
#include <iostream>
#include <sstream>
#include <vector>

int main() {
    // 1. Описываем LL(1)-совместимую грамматику (без левой рекурсии и факторизованную)
    std::istringstream input_grammar(
        "E -> T E_prime \n"
        "E_prime -> + T E_prime | ε \n"
        "T -> F T_prime \n"
        "T_prime -> * F T_prime | ε \n"
        "F -> ( E ) | id \n"
    );
    
    // Загружаем контекстно-свободную грамматику
    auto cfg = fsm::cfg_load(input_grammar);

    // 2. Строим таблицу предсказывающего анализатора
    // Указываем маркеры: эпсилон = "ε", конец ввода = "$"
    fsm::ll1::table_builder builder(cfg, "ε", "$");
    
    fsm::ll1::table<std::string> table;
    try {
        table = builder.build();
    } catch (const std::exception& e) {
        std::cerr << "Ошибка при построении таблицы: " << e.what() << '\n';
        return 1;
    }

    // 3. Красиво выводим полученную таблицу в консоль (в виде списка правил M[A, a] = ...)
    fsm::ll1::table_printer_settings print_settings;
    print_settings.format = fsm::ll1::table_format::rules_list;
    fsm::ll1::print_table(table, std::cout, print_settings);

    // 4. Проверяем корректность входной цепочки (id * id + id)
    std::vector<std::string> sequence = {"id", "*", "id", "+", "id"};

    std::cout << "\nЗапуск анализатора...\n";
    bool is_valid = fsm::ll1::accepts(
        table,               // Построенная таблица
        "E",                 // Стартовый нетерминал
        "ε",                 // Символ эпсилон (для удаления со стека)
        "$",                 // Символ конца (маркер дна стека и конца строки)
        sequence,            // Входная последовательность токенов
        &std::cerr,          // Поток для вывода синтаксических ошибок (опционально)
        &std::cout           // Поток для вывода пошаговой трассировки стека (опционально)
    );
    
    std::cout << "\nРезультат: " << (is_valid ? "Успех (Accepted)" : "Ошибка (Rejected)") << "\n";

    return 0;
}
```
